3. Technical Description:

i. Homepage Page

The Home Page serves as the central hub of the website, introducing users to the "Life on Land" initiative. It includes navigation links to all major pages and features a rotating logo, offering users a consistent and engaging entry point to explore further content.

HTML Techniques

  • The <div> tag is used to group layout elements such as the navbar, cards, and content blocks.
  • <section> is used to semantically divide different content sections for accessibility and SEO.
  • <a> and <img> tags are used for navigation and displaying visual content respectively.
  • <aside> is used for the statistics box to provide supplementary content beside the main article.
  • <footer> is used to provide the page's copyright and links.
  • Interactive areas such as buttons are created using anchor tags styled with CSS classes like .card-button.

Accessibility

  • The body uses a background-image and filter for visual depth.
  • The .navbar uses Flexbox (display: flex) to align logo and navigation links horizontally.
  • The .container and .card classes use Flexbox and box-shadow to create a responsive and modern layout for cards.
  • Animations like slideIn, fadeIn, and popIn add visual entrance effects to sections.
  • .footer uses position: fixed and visibility toggling via JavaScript to pop up when user scrolls to the bottom.
  • The .stats-box uses backdrop-filter and transparency to create a glassmorphic appearance.
  • Buttons and images have hover effects using :hover and transform transitions for interactivity.

Compliance

The Home Page complies with JANET regulations by using original content, proper external linking (e.g., logo and favicon from i.ibb.co), and ethical presentation. Navigation is clearly labeled and non-deceptive, and all external resources used (like icons or logos) are from reputable, open-hosted sources. The page avoids inappropriate or unsafe scripts, ensuring user security and accessibility.

Challenges and Lessons Learned

Challenges Encountered

Maintaining Design Consistency Across Pages

One of the biggest challenges was ensuring visual and structural consistency between multiple web pages (like Culture.html and WP.html). Each page had different content blocks (e.g., cards vs. infographics), but they had to feel like part of the same site. Matching layouts, padding, and colors across sections required fine-tuning in the shared CSS rules.

Navbar and Footer Positioning

Initially, the navigation bar and footer did not behave as expected - the navbar didn't remain visible during scroll, and the footer didn't animate in or out. This was resolved by integrating fixed positioning and JavaScript scroll detection to toggle the .visible class dynamically for the footer.

Image Alignment and Responsiveness

On smaller screens, infographics and images (e.g., the cartoon-style purification image in WP.html) either overlapped with text or were stretched. The fix involved adding flex-wrap, width limits, and max-width constraints in the .infographic-section and .action-box to ensure clean responsiveness.

Text Contrast and Readability

Due to the textured background image used in the body, some text sections became hard to read. This was resolved by layering semi-transparent white backgrounds (rgba) behind content areas like .content-wrapper and .call-to-action.

Lessons Learned

Component Reusability Matters

Reusing shared components (like navbars, footers, and CSS animation classes) reduces bugs and ensures consistency across pages.

Testing Across Devices is Crucial

Manual testing using different screen widths helped identify layout-breaking issues early. Using flexbox and media queries ensured the site looks good on both desktop and mobile.

JavaScript Adds Interactivity, But Needs Control

Implementing scroll-triggered animations and a sticky footer required carefully timed DOM queries and event listeners. Too many animations at once can hurt performance and user experience, so controlling visibility through scroll thresholds was key.
